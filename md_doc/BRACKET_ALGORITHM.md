# 자동 대진표 알고리즘

소스: `lib/session-bracket.ts`

---

## 생성 모드

| 모드 | 설명 |
|---|---|
| **STANDARD** | 전체 참가자를 풀로 대진 생성 |
| **TEAM_BATTLE** | A팀 vs B팀 구도 고정, 팀 내 같은 팀끼리 붙지 않음 |

---

## 입력 파라미터

| 파라미터 | 설명 |
|---|---|
| `courtCount` | 사용할 코트 수 (1~6) |
| `minGamesPerPlayer` | 1인 최소 경기 수 (1~5) |
| `separateByGender` | 남복/여복 분리 여부 |
| `relaxedMode` | 완화 모드 (제약 조건 일부 해제) |
| `generationMode` | STANDARD / TEAM_BATTLE |
| `teamAssignments` | 팀 배정 맵 `{ playerId: "A" | "B" }` |
| `fixedPairs` | 고정 파트너 쌍 `[playerId, playerId][]` |
| `seed` | 랜덤 시드 (없으면 Date.now()) |

## 1. 선수 점수 계산

### 급수 기본 점수

| 급수 | 점수 |
|---|---|
| S | 7 |
| A | 6 |
| B | 5 |
| C | 4 |
| D | 3 |
| E | 2 |
| 초심 | 1 |

### 나이 보정 (S~D 급수에만 적용)

| 나이 | 조정 |
|---|---|
| 60세 이상 | -3단계 |
| 50세 이상 | -2단계 |
| 40세 이상 | -1단계 |
| 40세 미만 | 없음 |

### 성별 보정 (통합 복식 모드에서만 적용)

- 여성: -1단계 (혼복 배정 시 팀 밸런스 보정 목적)
- 남성: 없음

> 보정된 점수가 팀전 밸런스 계산의 기준이 된다.

---

## 2. 라운드 생성 흐름

매 라운드마다 다음 순서로 진행된다.

```
1. 코트 배분 (풀별 경기 수 결정)
2. 출전 선수 선택
3. 매치 페어링 (팀 구성 최적화)
4. 히스토리 업데이트 (파트너/상대 기록)
5. 상태 업데이트 (경기수/휴식수/마지막 경기 라운드)
```

모든 선수가 `minGamesPerPlayer`를 달성하면 종료.

---

## 3. 출전 선수 선택 우선순위

라운드마다 출전할 선수를 아래 순서로 정렬해 상위 N명(코트수×4)을 선택한다.

1. **직전 라운드 휴식자 우선** (반드시 이번 라운드 출전)
2. 최소 경기 수 미달 선수 우선
3. 총 경기 수가 적은 선수 우선
4. 마지막 경기가 더 오래된 선수 우선
5. 랜덤 순서 (시드 기반)
6. 이름 가나다 순 (동점 시 결정)

> **strict 원칙**: 직전 라운드에 쉰 선수는 다음 라운드에 반드시 출전해야 한다.  
> 이 조건을 맞출 수 없을 때 에러 발생 → 완화 모드 제안.

---

## 4. 매치 페어링 점수 (낮을수록 좋음)

4명이 확정되면 3가지 팀 구성 조합 중 최적을 선택한다.

```
페어링 점수 =
  팀 점수차(밸런스갭) × 14        ← 핵심: 실력 균형
  + 파트너 반복 횟수 × 80         ← 같은 파트너 최대한 회피
  + 상대 반복 횟수 × 15           ← 같은 상대 최대한 회피
  + 고정 파트너 밸런스 보호 페널티   ← 고정 파트너 팀이 너무 강할 때 +20~40
  + 전체 점수 스프레드 × 3
  + 랜덤 바이어스 (미세 노이즈)
```

### 후보군 선택 방식

- 최저 점수 기준 +8 이내, 최대 5개 후보를 풀에 담고
- 그 중에서 랜덤 선택 → 매번 약간 다른 대진 생성

---

## 5. 통합 복식(STANDARD) 매치 구성 방식

남녀가 혼재할 때 아래 우선순위로 배정한다.

```
Phase 1: 혼복 (남+여 vs 남+여) 최대한 배정
Phase 2a: 남자 여유분 4명 이상 → 남남 vs 남남
Phase 2b: 여자 여유분 4명 이상 → 여여 vs 여여
Phase 2c: 수학적으로 어쩔 수 없는 잔여 → 혼합 배정
```

> 혼복 배정이 기본 우선이지만, 남녀 인원이 맞지 않으면 자연스럽게 단성 경기가 생긴다.

---

## 6. 팀 대항(TEAM_BATTLE) 특이점

- 매치는 항상 A팀 2명 vs B팀 2명 구조
- 팀 내 같은 팀끼리는 절대 붙지 않음
- 고정 파트너는 반드시 같은 팀 안에서만 설정 가능
- 남복/여복 분리 시: 같은 성별 + 같은 팀 풀에서만 페어링
- A팀과 B팀 인원이 다를 경우 경고 메시지 발생 (생성은 됨)

---

## 7. 고정 파트너 동작

- 고정 파트너로 지정된 두 선수는 매 라운드 항상 같은 팀에 배정
- 고정 파트너 쌍이 선택될 때는 반드시 둘 다 함께 선택됨 (한 명만 선택 불가)
- 고정 파트너가 있는 팀이 실력 차가 너무 심할 경우 추가 페널티 적용

---

## 8. 완화 모드(relaxedMode)

아래 조건이 충족 안 될 때 사용자에게 완화 모드 여부를 묻는다.

완화 모드에서 해제되는 제약:
- 직전 라운드 휴식자 반드시 출전 규칙
- 복수 코트 필요 조건 (requiredMatches > courtCount)

완화 모드에서도 유지되는 것:
- 밸런스 최적화
- 파트너/상대 반복 회피
- 고정 파트너 유지

---

## 9. 에러 케이스 정리

| 상황 | 에러 |
|---|---|
| 참가자 4명 미만 | 생성 불가 |
| 연속 휴식 발생 가능성 (인원 과다) | strict 모드에서 에러 → 완화 모드 제안 |
| 직전 휴식자를 코트 수 안에 넣을 수 없음 | strict 모드에서 에러 → 완화 모드 제안 |
| 최소 경기 수 달성 불가 | strict 모드에서 에러 → 완화 모드 제안 |
| 팀 대항인데 미배정 선수 존재 | 생성 불가 |
| 팀 대항인데 A팀 or B팀 2명 미만 | 생성 불가 |
| 고정 파트너가 다른 팀에 배정됨 | 생성 불가 |
| 남복/여복 분리인데 성별 미입력 선수 존재 | 생성 불가 |
