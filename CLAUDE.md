# 콕매니저 — Claude 협업 기준 문서

## 프로젝트 개요

배드민턴 클럽/소모임 총무·운영진을 위한 SaaS 운영 관리 프로그램.
회원 관리, 운동 일정 참석, 회비, 자동 대진표, 실시간 코트 배정 기능을 제공한다.

## 기술 스택

- **Framework**: Next.js 15 (App Router), React 19
- **DB**: PostgreSQL + Prisma ORM
- **Auth**: JWT (jose), 쿠키 기반 세션
- **배포**: Vercel (리전: icn1)
- **PWA**: 서비스워커 (`public/sw.js`), 빌드 시 버전 자동 갱신 (`scripts/update-sw-version.js`)
- **알림**: 텔레그램 봇 (`lib/telegram.ts`)

## 디렉토리 구조

```
app/
  api/          → API Route Handlers
  admin/        → 관리자 페이지 (로그인, 대시보드)
  join/         → 공개 가입 신청 페이지
  session/      → 공개 운동 일정 참석 페이지
components/
  dashboard/    → 대시보드 UI 컴포넌트 전체
lib/            → 공통 유틸, auth, telegram, prisma 등
prisma/         → DB 스키마 및 마이그레이션
public/         → 정적 파일 (sw.js, PWA 아이콘 등)
md_doc/         → 프로젝트 관리 문서
```

## ⛔ 절대 불변 원칙 — 유저 DB 불침범

**실서비스 유저의 DB 데이터는 어떤 이유로도 Claude가 임의로 수정·삭제·추가하지 않는다.**

- 코드 수정, 기능 구현, 버그 픽스 등 모든 작업에서 이 원칙이 1순위
- API Route, Prisma 쿼리, 스크립트 작성 시 항상 "이게 실유저 데이터를 건드리는가?" 먼저 확인
- 테스트용 데이터 조작은 반드시 `scripts/` 내 전용 스크립트로만, 유저가 직접 실행
- 마이그레이션도 마찬가지 — 스키마 변경은 가능하나 기존 데이터 변형은 유저 승인 없이 불가
- 이 원칙을 어기는 순간 서비스 전체가 무너진다

## 절대 건드리면 안 되는 로직

### 자동 대진 — 급수필터별(filter) 전용 코트 수 검증

`SessionBracketPanel.tsx`의 `levelModeErrors` useMemo 안에 **급수필터별(filter) 모드 전용** 코트 수 부족 검증이 있다.

규칙: 각 그룹의 필요 코트 수 = `Math.floor(그룹인원 / 4)`, 합산이 설정 코트 수를 초과하면 에러.
- 예) 7명 그룹 → 1코트, 9명 그룹 → 2코트 → 최소 3코트 필요. 2코트 설정 시 에러.
- 에러 시 "대진표 생성 / 대진표 저장 / 결과 포함 저장" 버튼 세 개 모두 비활성화.

**⚠️ 이 검증은 급수필터별(filter)일 때만 동작한다. 일반 자동대진(급수통합, levelMode="none") 및 동일급수별(separate) 로직은 절대 건드리지 않는다.**

### 정원 관리 — 파티 단위 원칙
운동 일정 참석 신청은 **파티(회원 + 동반 게스트) 단위**로 동작한다.
- 파티는 절대 쪼개지지 않음 (전원 참석 or 전원 대기)
- 정원 감소 시 마지막 신청 파티부터 전체 대기 이동
- 정원 증가 시 대기 첫 파티부터 전체 자동 승격
- 부분 승격/부분 강등 없음

## 개발 원칙

- 기존 안정 기능을 깨지 않는 것이 최우선
- 새 기능보다 회귀 복구 우선
- 불필요한 추상화, 미래 대비 설계 금지 — 지금 필요한 것만 구현
- 에러 처리는 실제로 발생 가능한 경우에만

## 모바일 디자인 원칙

- **무조건 컴팩트** — 불필요한 여백, 줄바꿈 금지
- 한 줄에 들어갈 정보는 한 줄 유지
- 의미 없는 중간 끊김 금지 (의미 단위로만 줄바꿈)
- 모바일에서는 정보 밀도와 화면 fit 우선

## 커밋 / 배포 규칙

- 커밋·푸시는 유저가 직접 (Claude는 코드 수정까지만)
- 배포: Vercel 자동 배포 (main 브랜치 push)
